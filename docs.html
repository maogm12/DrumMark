Drum Notation Guide

A text-first way to write professional drum scores. Fast, human-readable, and deterministic.

Overview

DrumMark is designed for musicians who want to write scores as fast as they type.

It follows four core principles: Human-first (readable without tools), Deterministic (consistent output), Validatable (strict error reporting), and Renderer-agnostic.

Example
# A basic groove
time 4/4
divisions 8

HH | x x x x x x x x |
SD | - - d - - - d - |
BD | b - - - b - - - |

Architecture

The compilation pipeline from source to notation.

The pipeline transforms DSL Source into an AST, then into a Normalized IR, which finally feeds the VexFlow Renderer or MusicXML Exporter.

Headers & Metadata

Defining the technical foundation of your score.

  • title, subtitle, composer: Score information.
  • tempo: Quarter-note BPM (default: 120).
  • time: Meter (e.g., 4/4, 6/8) - Required.
  • divisions: Grid resolution per measure - Required.
  • grouping: Beat structure (e.g., 2+2).
Example
title Funk Study
time 4/4
divisions 16
grouping 2+2

Tracks & Instruments

Mapping notes to specific drum kit components.

Each instrument is a track. Supported tracks include BD, SD, HH, RC, C, T1-T4, HF, and more. Use braces {} for routing scopes.

Example
HH | x - x - x - x - |
SD | - - d - - - d - |
BD | b - - - b - - - |
Score Result

Tokens & Rhythms

The atomic building blocks of your notation.

Tokens like d (hit), x (cymbal), and - (rest) are used to build patterns. Use . for dotted notes and / for halving duration. Square brackets [...] create rhythmic groups or tuplets.

Example
SD | d. d/ [2: d d d] d |
HH | x x x x |
Score Result

Modifiers & Techniques

Adding detail and articulation to your hits.

Apply :accent, :ghost, :flam, :open, :close, :roll, and more. Modifiers are validated against track legality.

Example
SD | d:accent - d:ghost - |
HH | d:open - d:close - |
Score Result

Combined Hits

Playing multiple parts simultaneously.

Use + to combine hits from different tracks or instruments at the same rhythmic position.

Example
| x+s - x+b - x+s - x+b - |
Score Result

Sticking

Hand assignments for complex patterns.

The ST track is used for R and L annotations. These align with notes at the same start position.

Example
SD | d d d d |
ST | R L R L |
Score Result

Repeats & Jumps

Structural navigation and alternative endings.

Supports repeat barlines |: :|, voltas |1., and navigation markers like @segno, @coda, @ds-al-coda.

Example
|: d d d d |1. d d d d :|2. d d d d |
Score Result

Multi-Measure Rests

Standard notation for extended periods of silence.

Use |--N--| to specify a rest spanning N measures.

Example
| --16-- |
Score Result

Inline Repeats

Quickly Duplicate measures in place.

Append *N to a measure to repeat it N times during compilation.

Example
HH | x x x x *4 |
Score Result

Validation

Ensuring musical and logical consistency.

The compiler strictly validates measure durations, grouping alignment, and modifier legality.

Example
# Error: cross boundary
time 4/4
grouping 2+2
divisions 4
