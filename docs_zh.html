鼓谱记谱指南

为现代鼓手设计的“文本优先”记谱方式。快速、易读、确定性强。

概览

DrumMark 为追求打字速度般的记谱体验的音乐家而设计。

它遵循四个核心原则：人类优先（无需工具即可阅读）、确定性（输出结果一致）、可验证性（严格的错误报告）和渲染器无关。

示例
# A basic groove
time 4/4
divisions 8

HH | x x x x x x x x |
SD | - - d - - - d - |
BD | b - - - b - - - |

头部与元数据

定义乐谱的技术基础。

元数据字段控制乐谱的身份和全局节奏逻辑：

  • title / subtitle / composer: 可选的文本字段，渲染在第一页。
  • tempo: 四分音符 BPM（默认 120）。影响播放和速度标记。
  • time: （必填）拍号（如 4/4, 6/8）。定义小节长度。
  • divisions: （必填）每小节的总网格单位。定义基础击打的“权重”。
  • grouping: 定义拍子结构（如 4/4 拍的 2+2）。控制默认重音和连杠逻辑。

title Funk Study
time 4/4
divisions 16
grouping 2+2

符号与节奏

记谱的基础原子单元。

原子符号代表特定的击打或休止。视觉形状根据符号和上下文推导：

  • d: 标准击打（根据轨道显示为实心圆或 X）。
  • -: 休止符。
  • x: 镲片或边击 (Cross-stick) 符号。
  • s / b / r / c: 军鼓、底鼓、叮叮镲和碎镲的简写。
  • o: 开启踩镲。
  • p: 踩镲脚踏或本地乐器击打。
  • g: 装饰音/鬼音 (Ghost note，带括号)。
  • t1 - t4: 通通鼓击打。

时值： 使用 . 表示附点（1.5倍），使用 / 表示时值减半（0.5倍）。

示例
SD | d. d/ [2: d d d] d |
HH | x x x x |
生成结果

修饰符与技巧

为击打添加细节和发音方式。

修饰符使用 : 附加（如 d:accent）。它们描述了鼓手的具体演奏技巧：

  • :accent: 重音，渲染为 >。
  • :ghost: 弱音/鬼音，带括号。
  • :flam: 装饰音 (Flam)，主音前的16分音符装饰。
  • :drag: 装饰音 (Drag)，主音前的两个16分音符装饰。
  • :roll: 滚奏/颤音，在音符符干上标注斜线。
  • :dead: 闷音 (Dead stroke)，小型 x 型音符头。
  • :open / :half-open: 踩镲开启/半开状态。
  • :bell: 叮叮镲镲帽 (Bell，渲染为圆点或 B)。
  • :choke: 镲片闷音击打（渲染为 +）。
  • :rim / :cross: 军鼓边击或横杠技巧。
示例
SD | d:accent - d:ghost - |
HH | d:open - d:close - |
生成结果

节奏组

连音符和自定义节奏细分。

使用方括号将项目组合到特定的时间跨度内：

  • [span: item1 item2 ...]: 在 span 个单位内平均分配项目。
  • [item1 item2 item3]: 连音符简写（跨度为 1）。
  • 嵌套连音符: 节奏组可以嵌套，用于实现如“三连音中的三连音”等复杂节奏。

组合击打

同时演奏多个声部。

使用 + 在同一节奏位置组合来自不同轨道或乐器的击打（例如 b+x 代表底鼓和踩镲同时击打）。

示例
| x+s - x+b - x+s - x+b - |
生成结果

粘滞 (Sticking)

为复杂模式分配左右手。

ST 轨道用于 R (右) 和 L (左) 标注。它们与同一起始位置的音符对齐，引导手部协调。

示例
SD | d d d d |
ST | R L R L |
生成结果

重复与跳转

结构化导航和替代结尾。

DrumMark 支持标准和复杂的乐谱导航：

  • 小节线: |: (开始重复), :| (结束重复), |1. (房子/结尾)。
  • 小节重复: % 符号重复整个前一小节。
  • 标记: @segno, @coda, @fine 标记特定位置。
  • 跳转: @to-coda, @da-capo (D.C.), @dal-segno (D.S.) 引导乐谱流向。
示例
|: d d d d |1. d d d d :|2. d d d d |
生成结果

多小节休止

长段沉默的标准记谱方式。

使用 |--N--| 指定跨越 N 个小节的休止。这会渲染为标准的单条多小节休止符号。

示例

内联重复

快速就地复制小节。

在小节边界附加 *N（例如 | d d d d *4 |），在编译期间将该特定小节重复 N 次。

示例

校验

确保音乐和逻辑的一致性。

编译器严格校验您的乐谱，以防止最终记谱中出现错误：

  • 小节时长: 击打和休止符的总和必须精确匹配 divisions。
  • 分组对齐: 音符不能跨越 grouping 中定义的拍子边界。
  • 合法性: 像 :open 这样的修饰符仅允许在踩镲轨道上使用。
示例
# Error: cross boundary
time 4/4
grouping 2+2
divisions 4
HH | d. d/ d d |
生成结果
