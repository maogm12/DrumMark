鼓谱记谱指南

为现代鼓手设计的"文本优先"记谱方式。

概览

DrumMark 为追求打字速度般的记谱体验的音乐家而设计。

DrumMark 让您可以用简单的文本来编写鼓谱。您写下的就是您想要听到的声音，系统会把它转换成专业的乐谱。

这个格式的设计原则是易于阅读——即使没有电脑，您也可以看懂 DrumMark 文档中的节奏。它也是可靠的：同样的输入总是产生同样的输出，您完全可以预期得到什么结果。

示例
title Basic Groove
tempo 120
time 4/4
divisions 8

HH | x x x x x x x x |
SD | - - d - - - d - |
BD | p - - - p - - - |

头部与元数据

定义乐谱的技术基础。

每首乐曲都以几个基本事实开头，这些信息告诉系统如何解读您的音乐。它们放在文档最顶部，在任何音符之前。

  • title / subtitle / composer: 乐曲的名称和作者信息。这些会印在乐谱上。
  • tempo: 四分音符 BPM。如不指定默认为 120。
  • time: 拍号（如 4/4、3/4、6/8）。必填。
  • divisions: 每小节的总网格单位。必填。数值越高节奏越精细——标准精度用 16，复杂切分用 32。
  • grouping: 定义小节内的节拍结构。控制节奏分组和连杠（beaming）。
    拍号推断的分组 (Inferred Grouping)
    2/4, 2/21+1
    3/4, 3/81+1+1
    4/42+2
    6/83+3
    9/83+3+3
    12/83+3+3+3

    对于不规则拍号如 5/8 或 7/8，必须显式指定 grouping（例如 grouping: 3+2）。

示例
title Metadata Demo
subtitle Subtitle
composer G Mao
tempo 96
time 4/4
divisions 8
grouping 2+2

HH | x x   x x   x x   x x |
SD | - -   d -   - -   d - |
BD | [2:p] - p   [2:p] - - |

轨道与乐器

将音符映射到具体的鼓组组件。

乐谱中的每一行代表一个乐器。您先写乐器名称，然后跟一个竖线，后面是那个乐器在这个小节中的所有音符。

名称演奏什么类别说明
HH踩镲——您用脚和鼓棒演奏的金属镲镲片
HF踩镲脚踏——您的脚跟或脚尖踩踏板的动作踏板
SD军鼓——底下有响线的那个主要的鼓鼓
BD / BD2底鼓——您用脚踩的大鼓，或双踩（两只脚）鼓
T1 - T4通通鼓——安装在底鼓上的鼓，从高到低鼓
RC / RC2叮叮镲——您通常用右手演奏的大镲镲片
C / C2碎镲——在重要节拍上重击的点缀镲片镲片
SPL / CHN水镲和中国镲——特殊效果镲镲片
CB / WB / CL牛铃、木鱼和击掌——额外的打击乐声音打击乐
ST左右手标注——显示哪只手演奏哪个音符（R = 右手，L = 左手）标注
| ... |没有轨道名表头的匿名轨道行路由上下文像 d、x 这样的裸上下文 token 默认路由到 HH。

匿名轨适合在不想每一行都重复写轨道名时，像 drum tab 一样快速记谱。您仍然可以用 SD:d、BD:d、ST:R 这类前缀把单个音符显式路由到指定轨道，也可以和 x+s+b 这样的组合击打混写。

符号与节奏

记谱的基础原子单元。

小节中的每个符号代表一次击打或一次休止。有些 token 会随轨道上下文变化，有些静态 magic token 则始终映射到规范里固定的乐器。

Token解析结果说明
d / D本轨普通击打 / 本轨重音击打使用当前轨道；在镲类轨道上会渲染成 X 音头。
-休止基础时值占一个 rhythmic slot。
x上下文相关击打在镲类上下文中是镲片击打，在鼓类上下文中是军鼓 cross-stick，在匿名轨默认是踩镲击打。
s / SSD:d / SD:d:accent静态军鼓 magic token。
b / BBD:d / BD:d:accent静态底鼓 magic token。
b2 / B2BD2:d / BD2:d:accent第二底鼓。
r / RRC:d / RC:d:accent叮叮镲快捷写法；裸 R 只有在 ST 中才表示 sticking。
r2 / R2RC2:d / RC2:d:accent第二叮叮镲。
c / CC:d / C:d:accent碎镲快捷写法。
c2 / C2C2:d / C2:d:accent第二碎镲。
t1 / T1T1:d / T1:d:accent高音通通鼓。
t2 / T2T2:d / T2:d:accent高中音通通鼓。
t3 / T3T3:d / T3:d:accent低中音通通鼓。
t4 / T4T4:d / T4:d:accent低音通通鼓。
o / OHH:d:open / HH:d:open:accent开镲击打。
spl / SPLSPL:d / SPL:d:accentSplash 镲。
chn / CHNCHN:d / CHN:d:accentChina 镲。
cb / CBCB:d / CB:d:accent牛铃。
wb / WBWB:d / WB:d:accent木鱼 / wood block。
cl / CLCL:d / CL:d:accent拍手。
p / P本轨踏板击打 / 本轨重音踏板击打匿名轨里通常回退为踩镲脚踏；在支持踏板的本地轨道中按本轨解析。

修饰符与技巧

为击打添加细节和发音方式。

在音符后面加冒号再写技巧名称，可以为您的击打添加额外的细节。modifier 是否合法取决于最终解析到的目标轨道，所以不是每个 modifier 都能用于所有 track。

轨道 / 家族允许的 Modifiers典型用途
所有击打轨道accent通用重音标记。
HHopen, half-open, ghost, drag, roll, dead踩镲开合状态与手部发音变化。
HFclose, accent踩镲脚踏闭合类动作。
RC, RC2bell, choke, drag, roll叮叮镲镲帽与镲片发音。
C, C2, SPL, CHNchoke敲击后立刻闷音。
SDrim, cross, flam, ghost, drag, roll, dead军鼓专属的边击、cross-stick、装饰音与滚奏。
T1 - T4flam, ghost, drag, roll, dead通鼓装饰音、滚奏与闷音击打。
BD, BD2roll, dead持续脚踏滚奏或闷音底鼓。
CB, WB, CLaccent除了重音外没有额外技巧 modifier。

语法：<token>:<modifier> 或 Track:d:<modifier>。可以链式写合法组合，例如 SD:d:rim:accent 或 RC:d:bell:accent。

  • 踩镲：HH | d:open - d:close - d:ghost - d:dead - |
  • 叮叮镲 / 其他镲片：RC | d:bell - d:drag - d:roll - - | 与 C | d:choke:accent - - - - - - - |
  • 军鼓：SD | d:accent - d:rim - d:cross - d:flam - |
  • 通鼓：T1 | d:ghost - d:drag - d:roll - d:dead - |
  • 底鼓：BD | d:roll - d:dead - d - d |
示例
title Articulation Techniques
tempo 96
time 4/4
divisions 16
grouping 2+2

HH | d:open - d:close - d:ghost - d:dead - |

RC | d:bell - d:drag - d:roll - - |

C  | d:choke:accent - - - - - - - |

SD | d:accent - d:rim - d:cross - d:flam - |

T1 | d:ghost - d:drag - d:roll - d:dead - |

BD | d:roll - d:dead - d - d |

节奏组

连音符和自定义节奏细分。

有时您需要一个不符合常规网格的节奏——比如在两个拍的空间里放三个音符。使用方括号将音符分组，然后将它们拉伸或压缩以适应您的时间跨度。

  • [note1 note2 note3]: 三个音符在一个拍的空间内均匀演奏。这是三连音的简写形式。
  • [3: note1 note2 note3]: 同样的意思——三个音符在三个单位的时间内演奏。3: 表示"把这些塞进三个槽位"。
  • [5: note1 note2]: 把两个音符塞进五个槽位——这会拉伸它们超过正常长度。
  • 嵌套节奏： 您可以把组放在组里面来实现非常复杂的模式，比如三连音里面的三连音。
示例
title Rhythmic Groups
time 4/4
divisions 8
grouping 4

HH | x x x x x x x x |
SD | - - [2: d d d] - - - - |
BD | b - - - b - - - |
生成结果

组合击打

同时演奏多个声部。

当您想让多个乐器同时演奏时，用加号把它们写在一起。这叫做组合击打。两个音符同时开始，演奏时长也相同。

比如，x+s 表示踩镲和军鼓一起演奏。您也可以组合三个或更多乐器：x+s+b 表示踩镲、军鼓和底鼓同时击打。

示例
title Simultaneous Hits
time 4/4
divisions 8
grouping 4

HH | x+s - x+b - x+s - x+b - |
SD | - - d+BD:d - - - - |
生成结果

左右手

为复杂模式分配左右手。

当您需要显示哪只手演奏哪个音符时，使用 ST 行。写 R 表示右手，L 表示左手。它们与上方的音符对齐，这样演奏者就知道该怎么做。

ST 标注对于复杂的节奏、爵士乐谱或手部动作很重要的音乐特别有用。

示例
title Hand Coordination
tempo 90
time 4/4
divisions 8
grouping 4

SD | d d d d d d d d |
ST | R L R L R L R L |
生成结果

重复与跳转

结构化导航和替代结尾。

DrumMark 支持所有标准的音乐重复和跳转方式：

  • 反复记号： |: 开始反复，:| 结束反复。当音乐到达结束记号时，它会返回到开始处。默认播放两次。
  • 跳过结尾： 反复之后可以有多个结尾。用 |1. 表示第一个结尾，|2. 表示第二个。演奏者先演奏结尾1，然后反复，再演奏结尾2。
  • 小节反复： 单独写一个 % 表示重复整个前一小节。用 %% 表示重复前两个小节。
  • 导航标记： 您可以用 @segno（segno记号）、@coda（coda记号）或 @fine（结束）标记位置。然后使用跳转如 @da-capo（返回开头）、@dal-segno（返回到segno）或 @to-coda（跳到coda）。
示例
title Repeat Structure
time 4/4
divisions 8
grouping 4

HH |: x x x x |1. x x x x :|2. x x x x |
生成结果

多小节休止

长段沉默的标准记谱方式。

当整个乐队需要休止多个小节时，使用多小节休止。写一条横线，中间放小节数：|--8--| 表示"休止8小节"。这会渲染为一个单一的、高大的休止符号，横跨所有八个小节。

示例
title Extended Rest
time 4/4
divisions 4
grouping 4

| --16-- |
生成结果

内联重复

快速就地复制小节。

当您需要同一小节连续出现多次时，您可以只写一次然后重复。在小节末尾写 *4 表示重复四次。这对于完全重复的模式很方便，比如一个重复的四小节乐句。

示例
title Measure Duplication
time 4/4
divisions 8
grouping 4

HH | x x x x x x x x *4 |
生成结果

校验

确保音乐和逻辑的一致性。

DrumMark 会检查您的音乐是否有错误，这样您就不会得到令人困惑或无法演奏的乐谱。系统会验证：

  • 小节长度： 小节中所有的音符和休止加起来必须正好等于规定的时长。如果您用了太多或太少，系统会告诉您。
  • 节拍边界： 音符不能"跨越"到下一个节拍区域。这可以保持节奏清晰易读。
  • 技巧规则： 有些技巧只在某些乐器上有效。比如，:open 只在踩镲上有意义——如果您把它写在军鼓上，DrumMark 会提醒您。
示例
title Boundary Crossing
time 4/4
divisions 8
grouping 2+2

HH | d. d/ d d |
生成结果
