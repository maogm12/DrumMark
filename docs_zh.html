鼓谱记谱指南

为现代鼓手设计的"文本优先"记谱方式。

概览

DrumMark 面向已经懂拍号、分组、装饰音和常见套鼓记谱的鼓手。

DrumMark 用纯文本描述套鼓谱面。您写的是节奏、分工和发音法，系统再把它编译成标准鼓谱。

它的目标不是替代乐理，而是把鼓手已经熟悉的拍号、分拍、重音、装饰音、反复和 sticking，用尽量短的文本稳定表达出来。

  • 注释：# 之后到行尾都是注释。
  • 空白： 空格和 Tab 只负责分隔 token，不改变节奏含义；HH|d-d-| 与 HH | d - d - | 等价。
  • 段落： 头部之后，空行分段。段落只影响排版换系统，不改变音乐时间线。
示例
title Basic Groove
tempo 120
time 4/4
divisions 8

HH | x x x x x x x x |
SD | - - d - - - d - |
BD | p - - - p - - - |

头部与元数据

定义乐谱的技术基础。

头部写在所有谱面内容之前，用来定义拍号、内部分拍和输入分辨率。

  • title / subtitle / composer: 乐曲的名称和作者信息。这些会印在乐谱上。
  • tempo: 四分音符 BPM。如不指定默认为 120。
  • time: 拍号（如 4/4、3/4、6/8）。必填。
  • divisions: 每小节的总网格单位。必填。数值越高节奏越精细——标准精度用 16，复杂切分用 32。
  • grouping: 定义小节内的节拍结构。控制节奏分组和连杠（beaming）。
    拍号推断的分组 (Inferred Grouping)
    2/4, 2/21+1
    3/4, 3/81+1+1
    4/42+2
    6/83+3
    9/83+3+3
    12/83+3+3+3

    对于不规则拍号如 5/8 或 7/8，必须显式指定 grouping（例如 grouping 3+2）。

  • 语法形式： 头部是 字段名 空格 值，不是 JSON，也不需要逗号。可写 time 4/4，也可写 time 4 / 4。
  • grouping 规则： 显式写出的每一组相加必须严格等于拍号分子。比如 7/8 可以写 grouping 3+2+2。
  • divisions 的意义： 它定义每小节输入网格总数。校验时所有 token 的时值总和必须刚好等于这个总网格。

grouping 2+2

SD | - -   d -   - -   d - |
BD | [2:p] - p   [2:p] - - |

符号与节奏

记谱的基础原子单元。

token 是 DrumMark 的最小记谱单位。它既可以表示击打，也可以表示休止，还可以携带时值修饰与路由信息。

-休止基础时值占一个 rhythmic slot。

修饰符与技巧

为击打添加细节和发音方式。

在音符后面加冒号再写技巧名称，可以为您的击打添加额外的细节。modifier 是否合法取决于最终解析到的目标轨道，所以不是每个 modifier 都能用于所有 track。

轨道 / 家族允许的 Modifiers典型用途
所有击打轨道accent通用重音标记。
HHopen, half-open, close, ghost, drag, roll, dead踩镲开合状态与手部发音变化。
HFclose, accent踩镲脚踏闭合类动作。
RC, RC2bell, choke, drag, roll叮叮镲镲帽与镲片发音。
C, C2, SPL, CHNchoke敲击后立刻闷音。
SDrim, cross, flam, ghost, drag, roll, dead军鼓专属的边击、cross-stick、装饰音与滚奏。
T1 - T4flam, ghost, drag, roll, dead通鼓装饰音、滚奏与闷音击打。
BD, BD2roll, dead持续脚踏滚奏或闷音底鼓。
CB, WB, CLaccent除了重音外没有额外技巧 modifier。

语法：<token>:<modifier> 或 Track:d:<modifier>。modifier 合法性看“最终解析到的物理轨道”，不是看它写在哪一行。可以链式写合法组合，例如 SD:d:rim:accent 或 RC:d:bell:accent。

  • 踩镲：HH | d:open - d:close - d:ghost - d:dead - |
  • 叮叮镲 / 其他镲片：RC | d:bell - d:drag - d:roll - - | 与 C | d:choke:accent - - - - - - - |
  • 军鼓：SD | d:accent - d:rim - d:cross - d:flam - |
  • 通鼓：T1 | d:ghost - d:drag - d:roll - d:dead - |
  • 底鼓：BD | d:roll - d:dead - d - d |

节奏组

连音符和自定义节奏细分。

group 用来表达不落在常规网格上的节奏压缩与拉伸，比如三连音、五连音，或者“两个音占三格”这种记法。

  • 基本语法：[span: item1 item2 ...]。省略 span 时，[a b c] 等价于 [1: a b c]。
  • 含义： 组内每个 item 的时值 = 当前槽位时值 × span / itemCount。
  • 拉伸组： 当 itemCount ≤ span 时，属于把音值拉长；只有结果能落在标准音值、附点音值或合法连音上时才允许。
  • 压缩组： 当 itemCount > span 时，2:1 与 4:1 这类会被视为普通细分，其它比值按 tuplet 处理。
  • 不支持的情况： 任何需要系统自动拆成 tie 才能成立的 group，都会直接报错。
  • 可嵌套： group 里可以再放 group，但极小音值不保证低于 64 分音符仍有良好支持。
  • [d d d]：省略 span 的最简写法，等价于 [1: d d d]。
  • [3: d d d]：3 个音分 3 格，常用于直接指定组宽。
  • [2: d d d]：3 个音压进 2 格，典型压缩组。
  • [3: d d]：2 个音占 3 格，属于拉伸组。
  • [5: d d]：2 个音占 5 格，只有结果能合法落谱时才允许。
  • [2: d d d d]：4 个音压进 2 格，会被当成普通细分而不是 tuplet。
  • [3: d [2: d d d] d]：嵌套 group，适合表达更复杂的内部细分。

组合击打

同时演奏多个声部。

组合击打用 + 连接。每个组成项在同一拍点开始，默认共享同一时值。

x+s 是最常见的 hat+snare，x+s+b 是三件同时击打；也可以写显式形式如 HH:d + SD:d。+ 两侧可留空格，也可以不留。

示例
title Simultaneous Hits
time 4/4
divisions 8
grouping 4

HH | x+s - x+b - x+s - x+b - |
SD | - - d+BD:d - - - - |

左右手

用于演奏标注、教学提示和 sticking 说明。

ST 轨主要用来补充“怎么打”，而不是“打什么”。在练习谱、教材、rudiment 分配、爵士 ride/snare 配手，或者需要明确 sticking 的段落里尤其有用。

写 R 表示右手，L 表示左手。它会附着到同一拍点上的所有实际音符；如果某个 ST 拍点没有任何实际音符，导出时会被忽略。

示例
title Hand Coordination
tempo 90
time 4/4
divisions 8
grouping 4

SD | d d d d d d d d |
ST | R L R L R L R L |
生成结果

重复与跳转

结构化导航和替代结尾。

结构性记号是全谱级别的，不是单轨私有信息。它们可以写在任意一轨，但会作用于同一全局小节。

  • 小节线： | 常规；|: 反复开始；:| 反复结束；|| 双小节线；|. 明确结束当前 volta bracket。
  • 空白有语义： || 表示“没有夹小节”的双线；| | 中间若有空白，则表示中间存在一个空小节。
  • Volta： |1. 是一房，|1,2. 表示共用结尾。volta 终止于新的 volta、:| 或 |.。
  • 小节反复： % 重复前 1 小节，%% 重复前 2 小节。它必须单独占满整个小节，且不能链式引用另一个反复小节。
  • 导航标记： 支持 @segno、@coda、@fine、@to-coda、@da-capo、@dal-segno、@dc-al-fine、@dc-al-coda、@ds-al-fine、@ds-al-coda。
  • 冲突规则： 同一小节最多一个 marker、最多一个 jump。v1 不允许嵌套反复或交叉反复。
示例
title Repeat Structure
time 4/4
divisions 8
grouping 1+1+1+1

BD | b - - - b - - - | % | %% |

SD | - - s - - - s - | - - s - - - s - | - - s - - - - - |

多小节休止

长段沉默的标准记谱方式。

多小节休止只接受 |--N--| 这一种写法，例如 |--8--|。数字两侧至少各要有一个 -，并且整个结构必须完整落在单个小节里。

示例
title Extended Rest
time 4/4
divisions 4
grouping 4

| --16-- |

内联重复

快速就地复制小节。

*N 写在小节末尾，表示把当前小节扩展成总共连续 N 个相同小节，而不是“再重复 N 次”。例如 *4 的结果是总共 4 小节。

示例

time 4/4
divisions 8
grouping 4

HH | x x x x x x x x *4 |

校验

确保音乐和逻辑的一致性。

DrumMark 的校验重点是“谱面逻辑必须唯一、时值必须闭合”。系统会验证：

  • 整小节时值： 每小节里所有 token 的精确时值总和必须刚好等于拍号与 divisions 共同定义的整小节长度。
  • grouping 边界： token 或整个 group 都不能跨越 grouping 定义的分拍边界。
  • modifier 合法性： modifier 是否允许，取决于最终解析目标轨。例如 :open 只能用于 hi-hat 语义。
  • 结构冲突： 冲突的反复、volta、marker/jump、非法 group，以及未知轨道或未知 token 都会直接报错。
