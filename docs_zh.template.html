快速入门

鼓谱记谱指南

为现代鼓手设计的“文本优先”记谱方式。快速、易读、确定性强。

概览

DrumMark 为追求打字速度般的记谱体验的音乐家而设计。

它遵循四个核心原则：人类优先（无需工具即可阅读）、确定性（输出结果一致）、可验证性（严格的错误报告）和渲染器无关。

架构

从源码到记谱的编译流水线。

流水线将 DSL 源码转换为 AST，然后转换为归一化的 IR，最后提供给 VexFlow 渲染器或 MusicXML 导出器。

头部与元数据

定义乐谱的技术基础。

  • title, subtitle, composer: 乐谱信息。
  • tempo: 四分音符 BPM（默认：120）。
  • time: 拍号（如 4/4, 6/8） - 必填。
  • divisions: 每小节的网格细分数 - 必填。
  • grouping: 拍子结构（如 2+2）。

轨道与乐器

将音符映射到具体的鼓组组件。

每种乐器都是一个轨道。支持的轨道 include BD, SD, HH, RC, C, T1-T4, HF 等。使用花括号 {} 开启路由作用域。

符号与节奏

记谱的基础原子单元。

使用 d（击打）、x（镲片）和 -（休止）等符号构建模式。使用 . 表示附点，使用 / 表示时长减半。方括号 [...] 用于创建节奏组或连音符。

修饰符与技巧

为击打添加细节和发音方式。

应用 :accent, :ghost, :flam, :open, :close, :roll 等。修饰符会根据轨道合法性进行验证。

组合击打

同时演奏多个声部。

使用 + 组合来自不同轨道或乐器在同一节奏位置的击打。

粘滞 (Sticking)

为复杂模式分配左右手。

ST 轨道用于 R and L 标注。它们与同一起始位置的音符对齐。

重复与跳转

结构化导航和替代结尾。

支持重复小节线 |: :|，房子 |1.，以及导航标记如 @segno, @coda, @ds-al-coda。

多小节休止

长段沉默的标准记谱方式。

使用 |--N--| 指定跨越 N 个小节的休止。

内联重复

快速就地复制小节。

在小节后附加 *N，在编译期间将其重复 N 次。

校验

确保音乐和逻辑的一致性。

编译器严格校验小节时长、分组对齐以及修饰符的合法性。

中间表示 (IR)

记谱背后的数据结构。

基于 JSON 的格式，明确定义了包含归一化节奏事件的头部、轨道和小节。

MusicXML 导出

与标准记谱工具的互操作性。

将您的 DrumMark 乐谱导出为 MusicXML，以便在 MuseScore、Sibelius 或 Finale 中打开。
